🎯 Estructura HTML Básica

Los cimientos de toda página web

Todo documento HTML sigue una estructura fundamental compuesta por tres secciones principales: <html>, <head> y <body>.

Entender esta estructura es esencial para crear páginas web bien formadas, accesibles y que funcionen correctamente en todos los navegadores.

Anatomía de un Documento HTML

Estructura básica de cualquier página web:

<!DOCTYPE html>
<html lang="es">

  <head>
    <meta charset="UTF-8">
    <title>Mi Página</title>
  </head>

  <body>
    <h1>Hola Mundo</h1>
  </body>

</html>

Componentes principales:

  • <!DOCTYPE html>: Declaración del tipo de documento
  • <html>: Elemento raíz que contiene todo
  • <head>: Metadatos e información no visible
  • <body>: Contenido visible de la página

El Elemento <html>

El elemento raíz que envuelve todo el contenido del documento:

<html lang="es">
  <!-- Todo el contenido aquí -->
</html>

Atributos importantes:

  • lang: Idioma del documento (esencial para accesibilidad y SEO)
  • dir: Dirección del texto (ltr = izquierda a derecha, rtl = derecha a izquierda)
<!-- Español -->
<html lang="es">

<!-- Inglés -->
<html lang="en">

<!-- Árabe (derecha a izquierda) -->
<html lang="ar" dir="rtl">
💡 Importancia: El atributo lang ayuda a los lectores de pantalla, herramientas de traducción y motores de búsqueda.

El Elemento <head>

Contiene metadatos, enlaces a recursos y configuración. No se muestra visualmente en la página:

<head>
• Metadatos
• Título de la página
• Enlaces a CSS
• Enlaces a JavaScript
• Información SEO

Elementos comunes en el <head>:

  • <title> - Título en la pestaña del navegador
  • <meta> - Metadatos (charset, viewport, descripción)
  • <link> - Recursos externos (CSS, fuentes, iconos)
  • <script> - JavaScript
  • <style> - CSS interno

Meta Tags Esenciales

Las etiquetas meta proporcionan información sobre el documento:

<head>
  <!-- Codificación de caracteres (SIEMPRE primero) -->
  <meta charset="UTF-8">

  <!-- Responsive design -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- SEO -->
  <meta name="description" content="Descripción de la página">
  <meta name="keywords" content="html, estructura, tutorial">
  <meta name="author" content="Tu Nombre">

  <!-- Título (REQUERIDO) -->
  <title>Mi Sitio Web</title>
</head>
⚠️ Importante: charset debe ser la primera etiqueta dentro del <head> para evitar problemas de codificación.

Importando Estilos CSS

Existen tres formas de incluir CSS en tu documento:

<head>
  <!-- 1. CSS Externo (RECOMENDADO) -->
  <link rel="stylesheet" href="css/styles.css">

  <!-- 2. CSS Interno -->
  <style>
    body { background: white; }
  </style>
</head>

<body>
  <!-- 3. CSS Inline (evitar cuando sea posible) -->
  <div style="color: red;">Texto</div>
</body>

Otros recursos comunes:

<!-- Fuentes de Google -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Roboto" rel="stylesheet">

<!-- Favicon -->
<link rel="icon" type="image/png" href="favicon.png">

JavaScript en el <head>

Los scripts pueden ir en el <head>, pero requieren atributos especiales:

<head>
  <!-- ❌ Bloquea el renderizado -->
  <script src="script.js"></script>

  <!-- ✅ Descarga en paralelo, ejecuta cuando termine -->
  <script src="script.js" async></script>

  <!-- ✅ Descarga en paralelo, ejecuta al final -->
  <script src="script.js" defer></script>
</head>

Diferencias clave:

  • Sin atributos: Bloquea el renderizado hasta que cargue y ejecute
  • async: Ejecuta en cuanto carga (no garantiza orden)
  • defer: Ejecuta después del HTML (mantiene orden)
💡 Recomendación: Usa defer para scripts que dependen del DOM. Usa async para scripts independientes (analytics).

El Elemento <body>

Contiene todo el contenido visible de la página:

<body>
• Texto y contenido
• Imágenes y multimedia
• Formularios
• Navegación
• Todo lo que el usuario ve
<body>
  <header>
    <h1>Mi Sitio Web</h1>
    <nav>...</nav>
  </header>

  <main>
    <article>...</article>
  </main>

  <footer>...</footer>

  <!-- Scripts al final del body -->
  <script src="app.js"></script>
</body>

¿Dónde Colocar los Scripts?

❌ En el <head> sin defer/async:
<head>
  <script src="app.js"></script>
</head>

Bloquea el renderizado

✅ Al final del <body>:
<body>
  <!-- contenido -->
  <script src="app.js"></script>
</body>

Carga más rápida

✅ En el <head> con defer:
<head>
  <script src="app.js" defer></script>
</head>

Mejor opción moderna - descarga en paralelo

Rutas y Paths

Las rutas pueden ser relativas o absolutas:

<!-- Rutas RELATIVAS (relativas al archivo actual) -->
<link href="style.css" rel="stylesheet"> <!-- Mismo directorio -->
<link href="css/style.css" rel="stylesheet"> <!-- Subdirectorio -->
<link href="../style.css" rel="stylesheet"> <!-- Directorio padre -->

<!-- Rutas ABSOLUTAS -->
<link href="/css/style.css" rel="stylesheet"> <!-- Desde la raíz del sitio -->
<link href="https://ejemplo.com/style.css" rel="stylesheet"> <!-- URL completa -->

Estructura de proyecto típica:

proyecto/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── app.js
└── images/
    └── logo.png

Restricciones Importantes

⚠️ Restricciones de Importación:
  • Los navegadores bloquean archivos de dominios externos sin CORS
  • Los scripts locales requieren un servidor (no funcionan con file://)
  • El orden de carga de scripts importa si hay dependencias
✅ Buenas Prácticas:
  • Minifica CSS y JS para producción
  • Usa rutas relativas para portabilidad
  • Agrupa recursos por tipo (css/, js/, images/)
  • Incluye charset y viewport en todas las páginas
  • Carga scripts al final o usa defer/async
  • Optimiza el orden: CSS en head, JS al final

✅ Pon a Prueba tus Conocimientos

Evalúa tu comprensión de la estructura HTML:

1. ¿Cuál es la diferencia entre el <head> y el <body>?

A) No hay diferencia, ambos muestran contenido
B) <head> contiene metadatos no visibles, <body> contiene el contenido visible
C) <head> es para títulos, <body> es para párrafos
D) <head> es opcional, <body> es obligatorio
Respuesta correcta: B) El <head> contiene metadatos, enlaces a CSS/JS y configuración que no se muestra visualmente. El <body> contiene todo el contenido visible de la página.

2. ¿Por qué es importante el meta charset="UTF-8"?

A) Hace que la página cargue más rápido
B) Define la codificación de caracteres para mostrar texto correctamente
C) Es solo para sitios en inglés
D) Cambia el color del texto
Respuesta correcta: B) charset="UTF-8" define la codificación de caracteres, permitiendo mostrar correctamente acentos, ñ, y caracteres especiales. Debe ser la primera etiqueta en el <head>.

3. ¿Cuál es la diferencia entre defer y async en scripts?

A) No hay diferencia, son sinónimos
B) defer ejecuta después del HTML (mantiene orden), async ejecuta cuando carga (no garantiza orden)
C) async es más lento que defer
D) defer solo funciona en Chrome
Respuesta correcta: B) defer descarga en paralelo pero ejecuta después del HTML (mantiene orden de scripts). async descarga y ejecuta inmediatamente cuando termina (no garantiza orden).

4. ¿Cuál es la diferencia entre rutas relativas y absolutas?

A) Las relativas son más rápidas que las absolutas
B) Relativas son relativas al archivo actual, absolutas desde la raíz del sitio o URL completa
C) Las absolutas solo funcionan en servidores
D) No hay diferencia práctica
Respuesta correcta: B) Rutas relativas (css/style.css, ../images/logo.png) son relativas a la ubicación del archivo actual. Rutas absolutas (/css/style.css, https://...) son desde la raíz del sitio o URL completa.

💪 Ejercicios Prácticos

Refuerza tus conocimientos con estos ejercicios:

Ejercicio 1: Construir Estructura HTML Completa

Objetivo: Crear un documento HTML bien estructurado desde cero

Requisitos:

  • Incluye <!DOCTYPE html> al inicio
  • Agrega <html lang="es">
  • En el <head>: charset UTF-8, viewport meta tag, título descriptivo
  • Agrega meta description y meta keywords para SEO
  • Enlaza un archivo CSS externo (css/styles.css)
  • Agrega un favicon
  • Incluye un script con defer al final del <head>
  • En el <body>: header, main, footer con contenido básico

Desafío Extra: Agrega Open Graph meta tags para redes sociales

Ejercicio 2: Organizar Estructura de Proyecto Multi-Página

Objetivo: Crear una estructura de carpetas organizada para un sitio de 3 páginas

Requisitos:

  • Crea carpetas: css/, js/, images/
  • Crea 3 páginas HTML: index.html, about.html, contact.html
  • Todas las páginas deben enlazar al mismo styles.css usando rutas relativas
  • Todas las páginas deben cargar el mismo script.js
  • Agrega una navegación en cada página que enlace a las otras
  • Verifica que todos los enlaces y rutas funcionen correctamente

Desafío Extra: Crea una estructura donde about.html está en una subcarpeta pages/ y ajusta todas las rutas

🎉 ¡Domina la Estructura HTML!

Puntos clave para recordar:

  • Todo documento HTML necesita <!DOCTYPE html>
  • <html lang="es"> es el elemento raíz - siempre incluye el atributo lang
  • <head> contiene metadatos, CSS y configuración (no visible)
  • <body> contiene todo el contenido visible
  • Siempre incluye charset y viewport meta tags
  • CSS va en el <head> (archivos externos preferidos)
  • JavaScript al final del <body> o en head con defer
  • Usa rutas relativas para archivos del proyecto
  • Organiza recursos en carpetas (css/, js/, images/)

¡Una estructura correcta es la base de todo sitio web exitoso!
