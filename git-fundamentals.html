Control de versiones para desarrolladores

Git es un sistema de control de versiones distribuido que permite rastrear cambios en tu código, colaborar con otros desarrolladores y mantener un historial completo de tu proyecto.

Es la herramienta más utilizada en desarrollo de software moderno y esencial para trabajar en equipo.

¿Qué es Git?

Git es un sistema de control de versiones que permite:

📜 Historial Completo: Guarda todos los cambios realizados en el código
👥 Colaboración: Múltiples desarrolladores trabajando en paralelo
🔄 Reversión: Volver a versiones anteriores cuando sea necesario
🌿 Ramificación: Trabajar en features sin afectar el código principal

Conceptos clave:

  • Repository (Repo): Carpeta del proyecto con historial Git
  • Commit: Snapshot (instantánea) de cambios guardados
  • Branch: Línea independiente de desarrollo
  • Remote: Versión del repositorio en un servidor (GitHub, GitLab)

Git Clone - Clonar Repositorios

Clonar copia un repositorio remoto a tu computadora local:

# Clonar desde GitHub/GitLab
git clone https://github.com/usuario/proyecto.git

# Clonar con un nombre diferente
git clone https://github.com/usuario/proyecto.git mi-carpeta

# Clonar una rama específica
git clone -b nombre-rama https://github.com/usuario/proyecto.git

¿Qué sucede al clonar?

  • Descarga todo el código del proyecto
  • Descarga el historial completo de commits
  • Crea una carpeta con el nombre del proyecto
  • Configura automáticamente el remote "origin"
💡 Tip: Solo necesitas clonar una vez. Después usarás git pull para actualizar.

Git Add - Preparar Cambios

El staging area es donde preparas los cambios antes de hacer commit:

# Agregar un archivo específico
git add index.html

# Agregar múltiples archivos
git add index.html style.css app.js

# Agregar todos los archivos modificados
git add .

# Agregar archivos por patrón
git add *.js # todos los archivos .js
git add css/ # toda la carpeta css

# Quitar del staging (sin perder cambios)
git reset archivo.js
⚠️ Importante: Los cambios no preparados (unstaged) no se incluirán en el commit.

Git Commit - Crear Snapshots

Un commit guarda permanentemente los cambios en el historial:

# Commit con mensaje
git commit -m "Agrega funcionalidad de login"

# Commit con mensaje detallado
git commit -m "Título del commit" -m "Descripción más larga"

# Add + commit en un solo paso (solo archivos tracked)
git commit -am "Actualiza estilos del header"

# Ver historial de commits
git log
git log --oneline # versión compacta

Buenas prácticas para mensajes de commit:

  • Usa verbos en presente: "Agrega" no "Agregado"
  • Sé conciso pero descriptivo
  • Explica el QUÉ y el POR QUÉ, no el CÓMO
  • Un commit = una funcionalidad o fix

Git Branch - Ramas de Desarrollo

Las ramas permiten trabajar en features sin afectar el código principal:

# Ver todas las ramas
git branch

# Crear una nueva rama
git branch feature-login

# Cambiar a una rama
git checkout feature-login

# Crear y cambiar a la rama en un solo paso
git checkout -b feature-login

# Eliminar una rama
git branch -d feature-login

# 1. Crear rama para nueva feature
git checkout -b feature-dashboard

# 2. Hacer cambios y commits en la rama
git add .
git commit -m "Implementa dashboard básico"

# 3. Volver a main
git checkout main

# 4. Fusionar la rama feature a main
git merge feature-dashboard

Git Push - Subir al Remoto

Push envía tus commits locales al repositorio remoto (GitHub, GitLab, etc.):

# Push a la rama actual
git push

# Primera vez que haces push de una rama nueva
git push -u origin nombre-rama

# Push a una rama específica
git push origin main

# Push de todas las ramas
git push --all

# Push forzado (¡usar con cuidado!)
git push --force
⚠️ Advertencia: Nunca uses --force en ramas compartidas (main, develop). Puede sobrescribir el trabajo de otros.

Git Pull - Actualizar desde Remoto

Pull obtiene y fusiona cambios del repositorio remoto:

# Obtener cambios de la rama actual
git pull

# Pull de una rama específica
git pull origin main

# Solo descargar sin fusionar
git fetch

# Ver qué cambió (después de fetch)
git log origin/main

Diferencia entre pull y fetch:

  • git pull: Descarga + fusiona automáticamente (fetch + merge)
  • git fetch: Solo descarga, no fusiona (más seguro)
💡 Buena práctica: Haz git pull antes de empezar a trabajar cada día para tener los últimos cambios.

Comandos Git Esenciales

# Ver estado de archivos
git status

# Ver diferencias antes de commit
git diff

# Ver historial de commits
git log --oneline --graph

# Ver información del repositorio remoto
git remote -v

# Descartar cambios en un archivo
git checkout -- archivo.js

# Ver quién modificó cada línea
git blame archivo.js

# Configurar usuario (primera vez)
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

🎉 ¡Domina Git!

Puntos clave para recordar:

  • git clone - Copia un repositorio remoto a tu máquina (solo una vez)
  • git status - Revisa qué archivos han cambiado
  • git add - Prepara archivos para el commit
  • git commit -m "mensaje" - Guarda cambios en el historial
  • git push - Envía commits al repositorio remoto
  • git pull - Obtiene cambios del remoto
  • git branch - Trabaja en features sin afectar main
  • git merge - Fusiona ramas cuando la feature está lista
  • Haz commits pequeños y frecuentes con mensajes descriptivos

¡Git es esencial para todo desarrollador moderno!
