🎯 Estructuras de Control en JavaScript

Controla el flujo de ejecución de tu código

Las estructuras de control permiten tomar decisiones y repetir acciones en tus programas.

Son fundamentales para crear lógica compleja y aplicaciones dinámicas.

if, else if, else

Ejecuta código basado en condiciones:

let edad = 20;

if (edad < 13) {
  console.log("Niño");
} else if (edad < 18) {
  console.log("Adolescente");
} else if (edad < 65) {
  console.log("Adulto");
} else {
  console.log("Senior");
}

// if simple sin else
if (temperatura > 30) {
  console.log("Hace calor!");
}

Estructura:

  • if: Condición principal (obligatorio)
  • else if: Condiciones adicionales (opcional, múltiples)
  • else: Caso por defecto (opcional)

switch Statement

Alternativa a múltiples if-else cuando comparas la misma variable:

let dia = "lunes";

switch (dia) {
  case "lunes":
    console.log("Inicio de semana");
    break;
  case "miércoles":
    console.log("Mitad de semana");
    break;
  case "viernes":
    console.log("Casi fin de semana!");
    break;
  case "sábado":
  case "domingo":
    console.log("Fin de semana!");
    break;
  default:
    console.log("Día regular");
}

Importante: No olvides el break o continuará ejecutando los siguientes cases (fall-through).

for Loop

Repite código un número específico de veces:

// Sintaxis: for (inicio; condición; incremento)
for (let i = 0; i < 5; i++) {
  console.log(`Iteración ${i}`);
}
// Imprime: 0, 1, 2, 3, 4

// Recorrer un array
const frutas = ["manzana", "pera", "uva"];
for (let i = 0; i < frutas.length; i++) {
  console.log(frutas[i]);
}

// Loop hacia atrás
for (let i = 10; i > 0; i--) {
  console.log(i);
}
console.log("¡Despegue!");

Partes del for: Inicialización, condición (mientras sea true), incremento/decremento.

while y do-while Loops

Repite mientras una condición sea verdadera:

// WHILE - verifica primero, ejecuta después
let contador = 0;
while (contador < 5) {
  console.log(contador);
  contador++;
}

// DO-WHILE - ejecuta primero, verifica después
let numero = 0;
do {
  console.log(numero);
  numero++;
} while (numero < 5);

// Ejemplo práctico
let password = "";
while (password !== "secreto") {
  password = prompt("Ingresa contraseña:");
}

Diferencia clave: do-while garantiza al menos una ejecución, while podría no ejecutarse nunca.

for...of y for...in

Formas modernas de iterar colecciones:

// FOR...OF - Itera VALORES de arrays
const colores = ["rojo", "verde", "azul"];
for (const color of colores) {
  console.log(color);
}
// Imprime: rojo, verde, azul

// FOR...IN - Itera KEYS de objetos
const persona = {
  nombre: "Ana",
  edad: 30,
  ciudad: "Madrid"
};
for (const key in persona) {
  console.log(`${key}: ${persona[key]}`);
}

Cuándo usar cada uno:

  • for...of: Arrays, strings, iterables (valores)
  • for...in: Objetos (keys/propiedades)

break y continue

Controla el flujo dentro de loops:

// BREAK - Sale completamente del loop
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    break; // Sale del loop
  }
  console.log(i);
}
// Imprime: 0, 1, 2, 3, 4

// CONTINUE - Salta a la siguiente iteración
for (let i = 0; i < 5; i++) {
  if (i === 2) {
    continue; // Salta el 2
  }
  console.log(i);
}
// Imprime: 0, 1, 3, 4

Casos de uso:

  • break: Encontrar un elemento y salir
  • continue: Saltar elementos que no cumplen condición

try...catch...finally

Manejo de errores para prevenir que tu app se rompa:

try {
  // Código que podría fallar
  const datos = JSON.parse(jsonString);
  console.log(datos);
} catch (error) {
  // Maneja el error
  console.log("Error al parsear JSON");
  console.error(error.message);
} finally {
  // Siempre se ejecuta (opcional)
  console.log("Proceso completado");
}

// Ejemplo práctico
try {
  const resultado = dividir(10, 0);
} catch (error) {
  mostrarMensajeError(error);
}

Bloques: try (intenta), catch (captura errores), finally (siempre ejecuta).

throw - Lanzar Errores

Puedes lanzar tus propios errores:

function dividir(a, b) {
  if (b === 0) {
    throw new Error("No se puede dividir entre 0");
  }
  return a / b;
}

try {
  const resultado = dividir(10, 0);
} catch (error) {
  console.log(error.message);
}

// Validación de edad
function validarEdad(edad) {
  if (edad < 0 || edad > 120) {
    throw new Error("Edad inválida");
  }
  return true;
}

Buena práctica: Usa throw para validaciones y errores personalizados con mensajes descriptivos.

Casos de Uso Comunes

  • if-else: Validaciones de formularios, permisos, estados
  • switch: Menús, estados de aplicación, días de semana
  • for: Procesar arrays, generar elementos, contadores
  • while: Esperar condiciones, juegos, input del usuario
  • for...of: Iterar arrays modernos, strings, Sets, Maps
  • for...in: Iterar propiedades de objetos
  • try-catch: API calls, parseo de JSON, operaciones de archivo
  • break/continue: Búsquedas, filtros, optimización
// Ejemplo real: validación de formulario
for (const campo of formulario) {
  if (campo.value === "") {
    throw new Error(`${campo.name} es requerido`);
  }
}

✅ Pon a Prueba tus Conocimientos

Evalúa tu comprensión de estructuras de control:

1. ¿Cuál es la diferencia entre while y do-while?

A) No hay diferencia
B) do-while ejecuta al menos una vez, while podría no ejecutarse
C) while es más rápido
D) do-while no existe en JavaScript
Respuesta correcta: B) do-while ejecuta el código primero y luego verifica la condición, garantizando al menos una ejecución. while verifica primero.

2. ¿Para qué sirve break en un loop?

A) Pausa el loop temporalmente
B) Sale completamente del loop
C) Reinicia el loop desde el inicio
D) Incrementa el contador
Respuesta correcta: B) break termina el loop inmediatamente y continúa con el código después del loop.

3. ¿Cuándo deberías usar for...of vs for...in?

A) for...of para arrays (valores), for...in para objetos (keys)
B) Son intercambiables
C) for...of es obsoleto
D) for...in es más rápido siempre
Respuesta correcta: A) for...of itera valores de arrays/iterables. for...in itera keys/propiedades de objetos.

4. ¿Qué hace el bloque finally en try-catch?

A) Solo se ejecuta si hay error
B) Solo se ejecuta si NO hay error
C) Se ejecuta siempre, haya o no error
D) Finaliza la aplicación
Respuesta correcta: C) finally siempre se ejecuta, haya o no error. Es útil para limpieza de recursos.

5. ¿Qué hace continue en un loop?

A) Sale del loop
B) Salta a la siguiente iteración
C) Pausa el loop
D) Reinicia desde el inicio
Respuesta correcta: B) continue salta el resto del código en esa iteración y continúa con la siguiente.

Ejercicio 1: FizzBuzz

Objetivo: Clásico ejercicio de programación

Requisitos:

  • Usa un for loop de 1 a 100
  • Si el número es divisible por 3, imprime "Fizz"
  • Si es divisible por 5, imprime "Buzz"
  • Si es divisible por 3 Y 5, imprime "FizzBuzz"
  • Si no, imprime el número

Desafío Extra: Conviértelo en una función que acepte el rango como parámetros

Ejercicio 2: Validador de Usuario

Objetivo: Crear un sistema de validación con try-catch

Requisitos:

  • Crea una función validarUsuario(nombre, edad, email)
  • Usa throw si nombre está vacío o edad < 18
  • Usa throw si email no contiene "@"
  • Envuelve la llamada en try-catch
  • Muestra mensajes de error apropiados

Desafío Extra: Agrega validación para que el nombre tenga al menos 3 caracteres
