🎯 Capítulo 1 - Introducción a JavaScript

El lenguaje de programación que da vida a la web

JavaScript es el lenguaje de programación más usado en el mundo, presente en prácticamente todos los sitios web modernos.

Permite crear experiencias interactivas, dinámicas y atractivas para los usuarios.

¿Qué es JavaScript?

JavaScript es un lenguaje de programación interpretado, dinámico y orientado a objetos que permite agregar interactividad a las páginas web.

Características principales:

  • Lenguaje de Alto Nivel: Fácil de leer y escribir para humanos
  • Interpretado: Se ejecuta directamente sin compilación previa
  • Dinámicamente Tipado: Las variables no tienen tipo fijo
  • Basado en Prototipos: Sistema de herencia flexible
  • Funciones de Primera Clase: Las funciones son valores
  • Event-Driven: Responde a eventos del usuario

JavaScript se ejecuta en el navegador (front-end) y también en el servidor (Node.js).

Historia de JavaScript

1995: Brendan Eich creó JavaScript en solo 10 días mientras trabajaba en Netscape Navigator.

  • 1995: Creación inicial con el nombre "Mocha", luego "LiveScript"
  • 1996: Renombrado a "JavaScript" por marketing (Java era popular)
  • 1997: Estandarizado como ECMAScript (ES1)
  • 2009: ES5 - La versión más compatible
  • 2015: ES6/ES2015 - Revolución moderna (let, const, arrow functions, clases)
  • 2016-2024: Actualizaciones anuales (ES2016, ES2017... ES2024)

Hoy JavaScript es el único lenguaje de programación que funciona nativamente en todos los navegadores.

JavaScript en el Navegador

Cada navegador tiene un motor de JavaScript que interpreta y ejecuta el código:

  • V8: Google Chrome, Edge, Opera, Node.js
  • SpiderMonkey: Mozilla Firefox
  • JavaScriptCore: Safari

Proceso de ejecución:

  • 1. El navegador descarga el archivo HTML
  • 2. Encuentra la etiqueta <script>
  • 3. Descarga el código JavaScript
  • 4. El motor lo parsea y compila a código de máquina
  • 5. Ejecuta el código línea por línea
  • 6. Puede modificar el HTML/CSS en tiempo real

¿Qué es el DOM?

El DOM (Document Object Model) es una representación en forma de árbol del documento HTML.

<html>
  <head>
    <title>Mi Página</title>
  </head>
  <body>
    <h1>Hola Mundo</h1>
    <p>Contenido</p>
  </body>
</html>

El DOM permite a JavaScript:

  • Acceder y leer elementos HTML
  • Modificar contenido y atributos
  • Agregar o eliminar elementos
  • Cambiar estilos CSS
  • Responder a eventos del usuario (clicks, teclas, etc.)

Forma 1: JavaScript Inline

Código JavaScript directamente en atributos HTML:

<button onclick="alert('¡Hola Mundo!')">
  Haz Click
</button>

<a href="#" onclick="console.log('Click'); return false;">
  Enlace
</a>

Ventajas:

  • Rápido para pruebas simples
  • No requiere archivo separado

Desventajas:

  • ❌ Código difícil de mantener
  • ❌ Mezcla HTML con lógica
  • ❌ No se puede reutilizar
  • ❌ Problemas de seguridad

Recomendación: Evitar en producción, usar solo para demos rápidas.

Forma 2: <script> Interno

Código JavaScript dentro de la etiqueta <script> en el HTML:

<!DOCTYPE html>
<html>
<head>
  <title>Mi Página</title>
</head>
<body>
  <h1 id="titulo">Hola</h1>
  <button id="btn">Click</button>

  <script>
    const btn = document.getElementById('btn');
    btn.addEventListener('click', function() {
      alert('¡Hola!');
    });
  </script>
</body>
</html>

Ventajas: Todo en un archivo, útil para scripts pequeños

Desventajas: No se puede reutilizar entre páginas, archivo HTML grande

Forma 3: <script src> Externo

JavaScript en archivo separado (recomendado para producción):

<!-- En el HTML -->
<script src="script.js"></script>

<!-- Con defer (recomendado) -->
<script src="script.js" defer></script>

<!-- Con async -->
<script src="script.js" async></script>

Diferencias importantes:

  • Sin atributos: Bloquea el parseo del HTML hasta cargar y ejecutar
  • defer: Descarga en paralelo, ejecuta después de parsear HTML (orden garantizado)
  • async: Descarga y ejecuta apenas esté listo (sin orden garantizado)

Recomendación: Usar defer para scripts que interactúan con el DOM.

Casos de Uso Comunes

  • Validación de formularios: Verificar datos antes de enviar al servidor
  • Interactividad: Menús desplegables, acordeones, tabs
  • Manipulación del DOM: Crear, modificar, eliminar elementos dinámicamente
  • Animaciones: Efectos visuales, transiciones, scroll effects
  • AJAX/Fetch: Cargar datos sin recargar la página
  • Single Page Applications (SPA): React, Vue, Angular
  • APIs del navegador: Geolocalización, notificaciones, almacenamiento local
  • Juegos en el navegador: Canvas, WebGL

JavaScript es fundamental para cualquier desarrollo web moderno.

✅ Pon a Prueba tus Conocimientos

Evalúa tu comprensión de JavaScript:

1. ¿Qué significa que JavaScript es un lenguaje "interpretado"?

A) Se ejecuta directamente sin compilación previa
B) Necesita ser traducido a otro lenguaje primero
C) Solo funciona con un intérprete humano
D) Es más lento que otros lenguajes
Respuesta correcta: A) JavaScript es interpretado porque el navegador ejecuta el código directamente línea por línea, sin necesidad de compilarlo a código de máquina previamente (aunque los motores modernos sí hacen JIT compilation internamente).

2. ¿Qué es el DOM?

A) Document Object Model - representación en árbol del HTML
B) Data Object Manager - gestor de bases de datos
C) Dynamic Online Method - método de conexión
D) Document Operation Module - módulo de operaciones
Respuesta correcta: A) DOM es Document Object Model, una representación en forma de árbol del documento HTML que permite a JavaScript acceder y manipular los elementos de la página.

3. ¿Cuál es la diferencia principal entre defer y async?

A) defer garantiza orden de ejecución, async no
B) async es más rápido que defer
C) defer solo funciona en Chrome
D) No hay diferencia, son sinónimos
Respuesta correcta: A) defer descarga el script en paralelo pero garantiza que se ejecutará después de parsear el HTML y en orden. async descarga y ejecuta apenas esté listo, sin garantizar orden.

4. ¿En qué año se creó JavaScript?

A) 1985
B) 1995
C) 2005
D) 2015
Respuesta correcta: B) JavaScript fue creado en 1995 por Brendan Eich en Netscape, originalmente en solo 10 días.

5. ¿Cuál es el motor de JavaScript de Google Chrome?

A) SpiderMonkey
B) JavaScriptCore
C) V8
D) Chakra
Respuesta correcta: C) V8 es el motor de JavaScript desarrollado por Google para Chrome, Edge, Opera y también es usado por Node.js.

💪 Ejercicios Prácticos

Refuerza tus conocimientos con estos ejercicios:

Ejercicio 1: Tu Primera Página con JavaScript

Objetivo: Crear una página HTML con JavaScript que muestre un mensaje al usuario

Requisitos:

  • Crea un archivo HTML con estructura básica
  • Agrega un botón con id="saludar"
  • Agrega un <script defer> que apunte a un archivo externo script.js
  • En script.js, selecciona el botón y agrega un event listener de click
  • Cuando se haga click, muestra un alert con "¡Hola desde JavaScript!"

Desafío Extra: En lugar de alert, modifica el contenido de un <h1> usando textContent

Ejercicio 2: Explorar el DOM

Objetivo: Familiarizarse con la consola del navegador y el DOM

Requisitos:

  • Abre cualquier sitio web (ej: google.com)
  • Abre DevTools (F12 o Cmd+Opt+I en Mac)
  • En la consola, escribe: document.querySelector('body')
  • Observa el objeto que devuelve
  • Prueba: document.body.style.backgroundColor = 'lightblue'
  • Experimenta cambiando colores y otros estilos

Desafío Extra: Usa document.querySelectorAll para seleccionar todos los enlaces y cambiarles el color
